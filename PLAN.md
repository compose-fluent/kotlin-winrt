- [x] Expect/actual support layout: support files generated with the opt-in `ExpectActualJvm` layout are written under `commonMain/kotlin` so projection support registries stay in the shared source set instead of leaking into an unowned third source root.
- [x] Narrow first slice: migrated non-generic interface/runtime-class method and property projection shape first; current implementation emits ordinary common interfaces plus JVM-side projection support objects instead of `expect`/`actual` interfaces, and emits simple non-generic runtime-class `expect`/JVM `actual` wrappers with matching RCW constructors that forward through the full public interface inheritance closure; interface/runtime-class forwarding covers inherited interface member forwarding with duplicate redeclaration suppression guarded by conflict checks, runtime-class member rows covered by inherited non-conflicting public interfaces with normalized raw interface names, matching member return/property types, method parameter names/types, no-exception HRESULT behavior, and property getter/setter shape; eligibility reuses the computed interface proxy closure for each decision point and permits member public ABI kinds inside the scalar/string/GUID/object/projected-interface/runtime-class boundary; unsupported ABI call plans, non-FFM ABI shapes, static-member types, setter-only-property projection types, async surfaces, and collection/custom mapped surfaces stay on the common fallback path instead of failing or silently dropping members during actual generation, while delegates, collections, async, structs, enums, activation, and generic closures remain queued for later target-specific call coverage.

- [x] Runtime fallback shrink: removed generated-projection scalar/mixed-shape `ComVtableInvoker.invokeArgs` overloads now covered by JVM actual FFM calls; retained runtime primitive shapes for `IUnknown`/`IInspectable`, events, collections, async, callbacks, value boxing, and temporary migration fallback.
